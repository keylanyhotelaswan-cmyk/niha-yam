## 1. Diagnosis (من التشغيل الفعلي)

| حقيقة | التفصيل |
| ----- | -------- |
| المصمم يغطي نوعين فقط | `receipt` · `kitchen` |
| العهدة / الرسائل التشغيلية | تطبع عبر Bridge بـ labels ثابتة — **خارج** Receipt Designer اليوم |
| بعض الحقول موجودة في الـ DB / snapshot وغير ظاهرة في المصمم | مثل `collected_by_name`, `created_at`, `collected_at` |
| حقول تشغيلية مهمة غير موصولة أصلًا | المنطقة · المندوب · آخر معدّل · تفصيل وسائل الدفع · رقم الوردية · الجهاز |
| تسمية `كاشير` مضللة | القيمة فعليًا = **منشئ الطلب** (`created_by`) |
| `datetime` الحالي | وقت الـ enqueue/`now()` وليس وقت إنشاء الطلب |
| مخاطر حية | `m6_build_order_print_payload` (OC identity) لا يضمّن `layout` ولا branding (`slogan` / عنوان الفرع / هاتف المطعم) — المعاينة قد تختلف عن الورق الحي |

## 2. Principles (مقفلة لهذا الـ Plan)

| ID | مبدأ |
| -- | ---- |
| **PF-1** | لا تغيير في بروتوكول Bridge / pairing / claim / ESC/POS pipeline |

## 6. Proposed — عهدة ورسائل (قرار Approve)

### خيار A (موصى به لهذا الإغلاق السريع)
- الإبقاء على render الثابت في Bridge للعهدة/الرسائل.
- تحسين **نصوص التسميات** فقط إن لزم (بدون مصمم).
- التأكد أن كل البيانات المهمة موجودة في الـ snapshot (موجودة تقريبًا).

### خيار B (إغلاق كامل للمصمم)
- فتح `PRINT_DOCUMENT_TYPES` ليشمل `shift_handover` و `ops_message`.
- نفس آلية sections/fields/labels.
- يتطلب عمل أكبر في UI + bake layout في enqueue — **خارج الحد الأدنى** إن أردنا سرعة الإغلاق.

**اقتراح الخطة:** **A الآن** · **B لاحقًا** فقط إذا طُلب تخصيص شكل العهدة/الرسائل.

حقول إضافية مقترحة للعهدة (حتى مع A — إن نقصت من الورق):
- اسم المستلم النهائي بعد الاعتماد  
- مرحلة التسليم (إنشاء / استلام) بتسمية أوضح  

حقول إضافية للرسائل:
- اسم المُرسل (`created_by_name`) — مفيد جدًا تشغيليًا  
- عنوان أوضح للمستهدف بالعربية

## 10. Success criteria

1. كل الحقول المعتمدة في §4–§5 اختيارية في المصمم مع تسميات عربية واضحة.  
2. الورق الحي يستخدم نفس `layout` المحفوظ (لا فرق معاينة/ورق بسبب payload).  
3. لا أسطر بعناوين بلا قيمة.  
4. إيصال دليفري يمكنه إظهار: عميل · هاتف · عنوان · منطقة · مندوب · منشئ · محصل · أوقات · دفع مفصّل.  
5. مطبخ يمكنه إظهار: طلب · نوع · منشئ · وقت · عميل · هاتف · منطقة · طاولة · أصناف · ملاحظة.  
6. `pnpm test:m6` أخضر + فحص يدوي Print Center.  
7. لا كسر لـ Operational Freeze خارج نطاق الطباعة المكتوب هنا.
